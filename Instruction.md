### 設計思維：
打造一個具有教育意義的防釣魚網站。
### 具體設計：
1. **網站名稱**：釣釣王
2. **網站目標**：教育用戶識別釣魚攻擊，提供防範措施和資源。
3. **網站結構**：
   - **首頁** (index.html)：
     - 彈出提示 step.1：說明這是一個釣魚測試網站，介紹釣魚攻擊的概念和危害。
     - 彈出提示 step.2：提供常見釣魚攻擊的特徵和範例。
     - 關閉提示後，展示首頁
     - 首頁模擬瀏覽器畫面，具備虛擬按鈕、網址欄等元素。每個元素可以點擊，讓使用者點擊指出如何識別釣魚網站的重點。
     - 密碼和登入欄位：當用戶可以輸入虛擬帳號和密碼，點擊登入時彈出提示。
     - 在頁面中設置虛假佈告欄，緊急提示使用者登入更換密碼。
     - 任何一個可點擊元件，都要給予警告提示。
   - **釣魚攻擊特徵**：
     - 位於首頁右側自動彈出，提供三步驟教學：
       - step 1.列出常見的釣魚攻擊特徵，如可疑的網址、拼寫錯誤、緊急語氣等。
       - step 2.提供範例圖片和說明，幫助用戶識別釣魚網站。
       - step 3.提供一個互動式測驗，讓用戶練習識別釣魚網站。
    - **防範措施**：
      - 位於首頁下方，提供防範釣魚攻擊的建議和資源，平時隱藏，點擊向上拉動展示。
      - 包括使用強密碼、啟用雙重驗證、定期更新密碼等。
      - 提供相關資源連結，如安全瀏覽器插件、官方安全網站等。
4. **express.js**：      
   - 使用 Express.js 搭建後端服務，提供靜態文件服務。
   - 設置路由處理首頁請求，返回 index.html。
   - 提供 API 接口，供前端獲取釣魚攻擊特徵和防範措施的數據。
   - 與 SQLite 數據庫交互，實現數據的增刪改查功能。
5. **SQLite**：
   - 使用 SQLite 數據庫存儲釣魚攻擊特徵和防範措施的數據。
   - 設計數據表結構，包含特徵描述、範例圖片、建議措施等字段。
   - 提供數據庫操作函數，實現增刪改查功能。
   - 與 Express.js 後端進行數據交互，提供 API 接口供前端調用。
   - 與首頁整合，實現動態數據展示，如釣魚攻擊特徵和防範措施的列表。
6. **前端技術**：
   - 使用 HTML、CSS 和 JavaScript 實現前端頁面。
   - 使用 Bootstrap 或其他 CSS 框架美化界面。
   - 使用 JavaScript 實現互動功能，如點擊提示、彈出警告等。
   - 使用 AJAX 獲取後端數據，動態更新頁面內容。
